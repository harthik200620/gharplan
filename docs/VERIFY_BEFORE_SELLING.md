# Data you MUST verify before selling

Every value below ships as an **indicative placeholder** so the product runs end-to-end.
**None of it is authoritative.** Verify each item with the right professional before you
quote a real client, imply Vastu compliance, or imply building-code compliance. Each is
flagged `TODO(human)` in the codebase.

> Product/legal framing is intentionally conservative — Vastu output is *"guidance, not
> certification"*, code output is a *"preliminary review"*, and every export footer says
> *"Indicative only. Not an approved/stamped drawing."* Keep those strings.

---

## 1. Rates, HSN codes & GST slabs — **a Quantity Surveyor**
- **Files:** `fixtures/rates/rates_seed.json` (engine source of truth), `fixtures/rates/rates_seed.sql`
  (Supabase mirror), generated by `engine/scripts/build_rates.py`.
- Material + labour rates are a Bengaluru base × per-city multiplier — **not market-checked**.
- HSN/SAC codes and GST % are best-guess (most finishing set to 18%). Verify each item's
  correct HSN/SAC and slab.
- **Action:** replace the `BASE` table and `CITY_MULTIPLIER` in `build_rates.py`, re-run it,
  and re-seed Supabase.

## 2. Building-code thresholds (bylaws / DCR) — **a registered architect / liaison**
- **File:** `fixtures/code_rules.json` (per state KA / MH / TG).
- Setback bands, FAR, ground-coverage %, min areas/dimensions, ceiling height, ventilation
  ratio, stair width and parking are indicative NBC-2016-ish values. Real rules vary by
  city (BBMP/KMC, DCPR/UDCPR, GHMC), zone, road width and plot size.
- **Also:** diagonal-facing setback mapping is approximate — see
  `engine/app/services/code_service.py` (`buildable_envelope`).
- **Action:** verify every number against the latest state DCR / municipal bylaws.

## 3. Vastu rule table — **a practising Vastu consultant**
- **File:** `fixtures/vastu_rules.json` (zones, weights, Brahmasthan rule).
- The seed encodes a common interpretation; traditions differ.
- **Brahmasthan strategy:** default is the classical 3×3 central cell (`grid_3x3`); the
  alternative is `area_rect_10pct`. Set via `GHARPLAN_BRAHMASTHAN`
  (`engine/app/config.py`). Confirm which your consultant prefers.

## 4. Billing & pricing — **founder/finance**
- **Files:** `web/lib/razorpay.ts` (`PRICING`), `packages/shared/src/constants.ts` (`BILLING`).
- ₹499/credit, ₹1,500 (5 plans) and ₹2,999 (unlimited) are placeholders.
- Subscriptions currently grant a **30-day access window** on payment; for true
  auto-recurring billing, create **Razorpay Plans/Subscriptions** and wire them in
  `/api/razorpay/*`.

## 5. Secrets & infra keys — **you**
- **Razorpay:** `RAZORPAY_KEY_ID`, `RAZORPAY_KEY_SECRET`, `RAZORPAY_WEBHOOK_SECRET`
  (`.env.example`). The app degrades gracefully (503) without them.
- **Supabase:** `NEXT_PUBLIC_SUPABASE_URL`, `NEXT_PUBLIC_SUPABASE_ANON_KEY`,
  `SUPABASE_SERVICE_ROLE_KEY`. Run `supabase/schema.sql` once.
- **Studio T&Cs:** each user sets their own legal terms in Settings (defaults are generic).

## 6. v2 generator templates — **an architect**
- **Files:** `fixtures/templates/*.json`, `engine/app/generator/`.
- Only `30x40_E` exists and is curated for ~30×40 East proportions; other facings return
  501 "coming soon". The linear scaler can drift Vastu zones for very different aspect
  ratios / setback bands. Keep behind `FEATURE_GENERATOR` until an architect authors more.
