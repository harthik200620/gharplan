# GharPlan

**A Vastu- & building-code-aware Design-to-Cost copilot for Indian residential design/build.**

Enter a plot or room layout → GharPlan checks it against **Vastu** rules and the
**National Building Code / local bylaws**, auto-generates an itemized **GST'd BOQ
directly from the room geometry**, and exports a client-ready **proposal (PDF)** and
**CAD (DXF)**. Built for independent interior designers and small (3–15 person)
design-build studios.

> The moat is the **integration through one canonical Plan schema**: the BOQ is
> generated from plan geometry, not typed by hand.

---

## Monorepo layout

| Path | What | Status |
|------|------|--------|
| `engine/` | Python 3.11+ FastAPI compute core (Vastu, code, **BOQ-from-geometry**, exports) | **runs + tested** |
| `packages/shared/` | Canonical Plan/BOQ TypeScript types + generated JSON Schema + constants | done |
| `fixtures/` | Plan samples, seed rates, Vastu/code/BOQ rule data | done |
| `web/` | Next.js 14 app (auth, wizard, canvas, editable BOQ, billing) | M3+ |
| `docker-compose.yml` | web + engine | M6 |

The canonical **Plan schema** (coordinates in metres, origin = plot SW corner,
`+x`=East, `+y`=North) is defined once as pydantic models in
`engine/app/models/plan.py`, exported to `packages/shared/plan.schema.json`, and
mirrored as TS types in `packages/shared/src/plan.ts`.

## Quick start — engine (the runnable core)

Requires Python 3.11+ (tested on 3.13). From `engine/`:

```bash
python -m venv .venv
.venv\Scripts\python -m pip install -r requirements.txt   # Windows
# source .venv/bin/activate && pip install -r requirements.txt   # macOS/Linux

# regenerate seed data + JSON Schema (committed, but reproducible)
python scripts/build_rates.py
python scripts/export_schema.py

# run the tests (the M1 gate)
python -m pytest -q

# serve the API
python -m uvicorn app.main:app --reload --port 8000
```

Then:

```bash
# normalize a plan (fills areaSqm / perimeterM / centroid / zone)
curl -X POST localhost:8000/plan/validate -H "Content-Type: application/json" \
  --data @fixtures/sample_plan_30x40_east.json

# generate a costed, GST'd BOQ from the same plan
curl -X POST localhost:8000/boq/generate -H "Content-Type: application/json" \
  -d "{\"plan\": $(cat fixtures/sample_plan_30x40_east.json), \"finishTier\": \"standard\"}"
```

### Engine API (v1)

| Method & path | Purpose | Status |
|---|---|---|
| `GET /health` | liveness | done |
| `POST /plan/validate` | validate + normalize a Plan to canonical form | done |
| `POST /boq/generate` | Plan (+ city, tier, edits) → itemized GST'd BOQ | done |
| `POST /vastu/check` | Plan → per-room pass/warn/fail + score + fixes | M2 |
| `POST /code/check` | Plan → NBC/bylaw report | M2 |
| `POST /export/{dxf,pdf,xlsx}` | exports | M2 |
| `POST /plan/generate` | v2 template generator (feature-flagged) | M5 |
